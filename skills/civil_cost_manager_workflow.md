## 역할 분담

| 역할 | 담당 | 하는 일 |
|---|---|---|
| 클로드 | AI | 아키텍처 설계, 작업 지시문 작성, 코드 리뷰 |
| 대호 | 프로젝트 오너 | 의사결정, 클로드↔미르 중계, localhost 검증 |
| 미르 | AI 코딩 에이전트 | 실제 코딩, Git 커밋/push |

> 클로드는 코드를 직접 작성하지 않는다.
> 클로드는 지시문을 작성 → 대호가 미르에게 전달 → 미르가 실행 → 대호가 결과 공유

---

## 커뮤니케이션 흐름

```
클로드 → 작업 지시문 작성
   ↓
대호 → 미르에게 전달
   ↓
미르 → 코딩 + 커밋 + 완료 보고
   ↓
대호 → localhost:8080 검증 후 클로드에게 결과 공유
   ↓
클로드 → 다음 단계 지시문 작성
```

---

## 작업 원칙
- 한 번에 한 단계만 진행
- 각 단계는 명확한 산출물 포함
- 미르 완료 보고 확인 후 다음 단계 진행
- 테스트 없이 push 금지
- .env 파일 커밋 금지
- **매 커밋마다 workflow.md 진행 상태 체크리스트 업데이트 필수**
  - 완료된 항목: `- [ ]` → `- [x]`
  - 새로 추가된 예정 작업은 해당 섹션에 추가
